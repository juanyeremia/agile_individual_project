Introduction

Throughout this project I used Claude (by Anthropic) as an AI assistant to help guide my learning and implementation of HTML, CSS, and JavaScript. This page documents how I used AI it, what it produced, the corrections I made, and a critical evaluation of its limitations.

Use Case 1: Overall Theming System

Prompt

I wanted to have a consistent look and across all four pages of the project. I consulted with Claude with the initial concept: "I want to have consistent UI elements across the entire webpage. What would be the best way to do this? I was thinking like a header at the top, a navbar that has position sticky so it will stop at the top and follow the scroll, and the content below it as a card." I had a barebones idea in mind and wanted guidance on how to structure it properly.

What Claude Produced

Claude agreed with the general concept and extended it by suggesting a color theming system to keep UI elements visually consistent. It proposed a Forest Terminal color palette, with deep green, emerald, and pale green tones, and introduced me the use of CSS custom properties to implement it. Specifically, it suggested defining all colors once in a :root block using CSS variables like --forest-nav, --forest-accent, and --forest-accent-lt, then referencing them throughout the CSS using var().

What I Changed

Claude initially suggested defining the :root variables inside tutorial.css. I immediately thought that it suggested to do this for each page's CSS. I thought that this would limit the variables to one specific page only, making them inaccessible to the other three pages. I decided it would be more efficient to have a general CSS file, general_structure.css, a shared stylesheet that will loaded by all four pages. This way the color system is defined once and available everywhere, while each page's CSS file only contains styles unique to that page. This decision reflects the separation of concerns principle covered in the CSS lecture.
